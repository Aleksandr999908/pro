# Информация о времени обучения

## Почему обучение долгое?

1. **CPU вместо GPU**: Обучение на CPU в 10-50 раз медленнее, чем на GPU
2. **Большой датасет**: 6097 кадров - это много данных для обработки
3. **Много эпох**: В стандартной конфигурации 50 эпох
4. **Большая модель**: EfficientNet требует много вычислений

## Время обучения (примерно):

### На CPU:
- **Быстрая конфигурация** (5 эпох, MobileNet): ~30-60 минут
- **Стандартная конфигурация** (50 эпох, EfficientNet): ~5-10 часов

### На GPU:
- **Быстрая конфигурация**: ~5-10 минут
- **Стандартная конфигурация**: ~1-2 часа

## Текущая конфигурация

**Запущено**: Быстрая версия (config_fast.yaml)
- Модель: MobileNetV3 (легче и быстрее)
- Эпох: 5 (вместо 50)
- Батч: 16 (меньше для скорости)

**Ожидаемое время**: 30-60 минут на CPU

## Что можно сделать:

1. **Подождать** - обучение идет в фоновом режиме
2. **Использовать GPU** - если есть NVIDIA GPU, установите CUDA
3. **Уменьшить данные** - обучить только на части датасета для теста

## Проверка прогресса

После завершения обучения:
- Модель будет в `models/checkpoints/best_model.pth`
- Логи обучения можно просмотреть в консоли

## Следующие шаги после обучения:

1. Экспорт модели в ONNX:
```bash
python scripts/export_onnx.py \
    --model models/checkpoints/best_model.pth \
    --config models/config_fast.yaml \
    --output models/fgc_day.onnx
```

2. Запуск API сервиса:
```bash
python service/app.py
```
